트리 자료구조의 필요성

  • 트리 자료구조는 결리스트의 검색 시, 노드의 처음부터 찾아가야하는 단점을 보완했음. 이진탐색의 장점을 활용함.

트리 자료구조 실생활 예시

  • 회사나 정부의 조직도
  • 나라, 지방, 시•군별, 계층적인 데이터의 저장
  • 인덱스
  • DOM Tree

용어

  • 루트노드 : 가장 꼭대기 노드.
  • 노드 : 트리 구조에서의 기본 데이터 저장 단위.
  • 리프 노드 : 차수가 0인 노드, 즉 맨 끝에 달린 노드를 말함.

트리 용어

  • 차수 : 선택한 노드의 부속 트리의 개수를 말함.
  • 트리의 차수 : 전체 트리의 최대 차수를 말함.
  • 부모 : 부속 트리를 가진 노드.
  • 자식 : 자신의 바로 상위 노드에 속하는 부속노드.
  • 형제 : 부모가 같은 자식 노드.
  • 조상 : 노드의 부모 노드들의 총 집합.
  • 자손 : 노드의 부속 트리에 있는 모든 노드들.
  • 레벨 : 루트 노드들로부터의 깊이.
  • 트리의 깊이 : 트리에 속한 노드의 최대 레벨.

필요한 메소드

  • get
  • add
  • remove

이진트리의 정의 및 조건

  • 정의 : 자식노드를 최대 2개까지 가질 수 있는 트리를 말한다.
  • 조건 1) 루트 노드를 중심으로 두 개의 서브 트리로 나뉘어진다.
  • 조건 2) 나뉘어진 두 서브 트리도 모두 이진 트리어야 한다.
  • 완전이진트리의 조건 : 마지막 레벨을 제외한 모든 레벨의 node가 완전히 채워져 있으며 마지막 레벨의 node들이 왼쪽부터 채워져 있는 구조를 말한다

이진트리의 활용 예 : 수식트리

  • 정의 : 이진트리를 이용해서 수식을 표현해 놓은 것.
  • 활용 : 수식 계산에 활용할 수 있는데, log(n)이 걸리므로 기존의 스택계산기보다 시간복잡도 면에서 더 나은 결과를 얻을 수 있다. 이진트리의 후위순회법 알고리즘을 활용하여 계산을 해주는 것이다.

이진탐색트리의 조건

  • 1) 이진탐색트리의 노드에 저장된 키는 유일해야 한다.
  • 2) 루트노드의 키가 왼쪽 서브트리를 구성하는 어떠한 키보다 커야한다.
  • 3) 루트노드의 키가 오른쪽 서브트리를 구성하는 어떠한 키보다 작아야 한다.
  • 4) 왼쪽과 오른쪽 서브 트리도 이진 탐색 트리이다.
